# Техническое Задание
## Система внешнего мониторинга ПК по WiFi

**Версия:** 1.0
**Дата:** 11.03.2026

---

## 1. Общее описание

Система состоит из двух компонентов:
- **Сервер** — приложение на ПК, собирающее данные из MSI Afterburner/RivaTuner и раздающее их по HTTP/WebSocket
- **Клиент** — устройство на ESP8266 с LCD-дисплеем, отображающее данные в реальном времени

---

## 2. Серверная часть (ПК)

### 2.1 Технологии
- **Язык:** Python 3.x или C# (.NET)
- **Протокол:** HTTP REST + WebSocket
- **Интеграция:** MSI Afterburner через MAHM Shared Memory (файл-маппинг `MAHMSharedMemory`)

### 2.2 Сбор данных

Считывать все доступные датчики через MAHM Shared Memory:

| Категория | Параметры |
|-----------|-----------|
| CPU | температура, загрузка (%), частота (MHz) |
| GPU | температура, загрузка (%), частота ядра/памяти, VRAM usage |
| FPS | текущий, минимальный, средний, 1% Low (RivaTuner) |
| Память | RAM usage (MB/%), VRAM usage |
| Питание | CPU Power (W), GPU Power (W), TDP % |
| Система | скорость вентиляторов (RPM), напряжения |

### 2.3 API

```
GET  /api/sensors          — список всех доступных датчиков
GET  /api/data             — текущие значения всех датчиков (JSON)
GET  /api/templates        — список шаблонов отображения
POST /api/templates        — создать/обновить шаблон
WS   /ws/data              — стрим данных (интервал: настраивается, по умолчанию 1с)
```

### 2.4 API привязки устройств

```
POST /api/bind              — запросить привязку устройства
POST /api/bind/confirm      — подтвердить ребайнд (если устройство занято)
DELETE /api/devices/{id}    — отвязать устройство
```

Тело запроса `/api/bind`:
```json
{ "device_id": "esp8266_A4B1C2" }
```

### 2.5 GUI приложения (настройка)

Десктопное окно (PyQt / WPF / Electron):

**Вкладка "Мониторинг"**
- Список активных датчиков с текущими значениями
- Индикатор состояния подключённых ESP-клиентов

**Вкладка "Шаблоны"**
- Визуальный редактор: выбор датчиков из списка, порядок отображения, метки, единицы измерения
- Настройка интервала обновления (100мс — 5000мс)
- Привязка шаблона к конкретному клиенту (по IP или MAC)
- Сохранение/загрузка шаблонов (JSON-файлы)

**Вкладка "Устройства"**

Список "Обнаруженные в сети":
- Таблица: `Имя устройства | MAC | Дисплей | Статус | Действие`
- Статусы:
  - Зелёный: **Привязано (этот ПК)** — активное устройство
  - Замок: **Привязано (DESKTOP-OLD)** — занято, доступна кнопка "Перепривязать"
  - Серый: **Свободно** — доступна кнопка "Привязать"

Список "Мои устройства" (привязанные к текущему ПК):
- Таблица: `Псевдоним | MAC | Дисплей | Последний онлайн | Шаблон | Действия`
- Действия: переименовать, сменить шаблон, отвязать

Кнопки панели:
- "Найти устройства" — запустить активный скан
- Индикатор: "Последнее сканирование: HH:MM:SS"

**Вкладка "Сеть"**
- Отображение IP сервера (для ввода в ESP)
- Порт HTTP/WS (по умолчанию: 8080)
- Список подключённых клиентов

## 3. Клиентская часть (ESP8266 + LCD)

### 3.1 Аппаратура
- **МК:** ESP8266 (NodeMCU / Wemos D1 Mini)
- **Дисплей:** I2C LCD 20×4 или 16×2 (HD44780 + PCF8574) **или** SSD1306 OLED 128×64

### 3.2 Прошивка (Arduino / PlatformIO)

#### Режим AP (первый запуск / нет подключения)
1. ESP поднимает точку доступа: `PC-Monitor-XXXX` (без пароля или с дефолтным паролем `12345678`)
2. Запускает DNS-сервер (captive portal) — любой запрос → редирект на `192.168.4.1`
3. Web-интерфейс настройки:
   - Сканирование и выбор WiFi сети
   - Ввод пароля WiFi
   - Ввод IP и порта сервера ПК
   - Кнопка "Сохранить и подключиться"
4. Настройки сохраняются в EEPROM/LittleFS

#### Режим клиента (нормальная работа)
1. Подключение к WiFi из сохранённых настроек
2. WebSocket-подключение к серверу ПК
3. Получение данных и отрисовка по активному шаблону
4. Heartbeat: если нет данных >10с — отображение ошибки на LCD
5. При потере WiFi (>30с) — переход в режим AP

#### Отрисовка на LCD
- Поддержка нескольких "страниц" с автопереключением (интервал настраивается)
- Каждая строка = один датчик из шаблона
- Формат строки: `[метка] [значение] [ед.изм]`, например `CPU: 65°C`
- Символы-иконки для часто используемых датчиков (только SSD1306)

### 3.3 Физическая кнопка (опционально)
- Одна кнопка: переключение страниц вручную
- Удержание 5с: сброс настроек и переход в режим AP

## 4. Механизм обнаружения устройств

### 4.1 Со стороны ESP8266 (beacon)

ESP8266 в режиме клиента периодически рассылает UDP-broadcast пакет каждые **5 секунд**:

```json
{
  "type": "pc_monitor_beacon",
  "device_id": "esp8266_A4B1C2",
  "mac": "A4:B1:C2:D3:E4:F5",
  "display": "LCD 20x4",
  "firmware": "1.0.0",
  "bound_to": "DESKTOP-GAMING",
  "bound_pc_id": "uuid-1234-abcd"
}
```

- `bound_to` / `bound_pc_id` — пустые строки если устройство не привязано
- Порт: **UDP 45454** (фиксированный)
- Параллельно регистрирует mDNS-сервис `_pcmonitor._tcp` как резервный метод
- В режиме AP beacon не рассылается

### 4.2 Со стороны сервера (активный скан)

При запуске и по кнопке "Найти устройства" сервер:
1. Отправляет UDP-broadcast `255.255.255.255:45454` с пакетом-запросом:
   ```json
   { "type": "discover_request" }
   ```
2. Слушает ответы 3 секунды
3. Параллельно слушает входящие beacon-пакеты постоянно в фоне

## 5. Привязка устройства (Binding)

### 5.1 Идентификаторы

| Сущность | Идентификатор |
|----------|---------------|
| ESP8266 | MAC-адрес (`device_id`) |
| ПК-сервер | `pc_id` = UUID, генерируется при первом запуске, хранится в `server_config.json` |
| Имя ПК | `pc_name` = hostname ОС (`socket.gethostname()`) — отображаемое имя |

### 5.2 Хранение привязки

**На ESP8266** (LittleFS, `binding.json`):
```json
{
  "bound": true,
  "pc_id": "uuid-1234-abcd",
  "pc_name": "DESKTOP-GAMING",
  "bound_at": 1741689600
}
```

**На сервере** (локально, `devices.json`):
```json
{
  "devices": [
    {
      "device_id": "esp8266_A4B1C2",
      "mac": "A4:B1:C2:D3:E4:F5",
      "display": "LCD 20x4",
      "alias": "Монитор на столе",
      "active_template": "Gaming",
      "last_seen": 1741689600
    }
  ]
}
```

### 5.3 Сценарии привязки

#### Устройство свободно (не привязано)

```
Сервер обнаружил ESP → bound_to пуст
  └→ GUI показывает кнопку "Привязать"
       └→ POST /api/bind
            └→ Сервер отправляет команду на ESP:
               { "type": "bind", "pc_id": "...", "pc_name": "DESKTOP-GAMING" }
                    └→ ESP сохраняет binding.json
                         └→ Ответ "OK" → устройство появляется в списке привязанных
```

#### Устройство привязано к другому ПК (перепривязка)

```
Сервер обнаружил ESP → bound_to = "DESKTOP-OLD"
  └→ GUI показывает устройство с меткой [замок] "Привязано к DESKTOP-OLD"
       └→ Пользователь нажимает "Привязать"
            └→ GUI показывает диалог:

  ┌─────────────────────────────────────────────────────┐
  │  Устройство уже привязано                            │
  │                                                      │
  │  ESP8266 [esp8266_A4B1C2] привязан к ПК:            │
  │  "DESKTOP-OLD"                                       │
  │                                                      │
  │  Для перепривязки к этому ПК нажмите "Подтвердить". │
  │  Устройство будет отвязано от предыдущего ПК.        │
  │                                                      │
  │           [Отмена]    [Подтвердить привязку]         │
  └─────────────────────────────────────────────────────┘

            └→ Пользователь подтверждает
                 └→ POST /api/bind/confirm { "device_id": "...", "force": true }
                      └→ Сервер отправляет ESP:
                         { "type": "rebind", "pc_id": "...", "pc_name": "DESKTOP-GAMING" }
                              └→ ESP перезаписывает binding.json
                                   └→ Устройство привязано к новому ПК
```

#### Устройство привязано к текущему ПК (повторное подключение)

```
Сервер обнаружил ESP → pc_id совпадает с собственным
  └→ Автоматически принимается без диалогов
       └→ Устройство добавляется в список активных клиентов
```

## 6. Форматы данных

### 6.1 JSON данных датчиков

```json
{
  "timestamp": 1741689600,
  "sensors": {
    "cpu_temp":    { "value": 65.0,  "unit": "°C",  "label": "CPU Temp" },
    "gpu_temp":    { "value": 72.0,  "unit": "°C",  "label": "GPU Temp" },
    "cpu_load":    { "value": 45.0,  "unit": "%",   "label": "CPU Load" },
    "gpu_load":    { "value": 98.0,  "unit": "%",   "label": "GPU Load" },
    "fps":         { "value": 120.0, "unit": "FPS", "label": "FPS" },
    "fps_1low":    { "value": 87.0,  "unit": "FPS", "label": "1% Low" },
    "ram_used":    { "value": 14.2,  "unit": "GB",  "label": "RAM" },
    "gpu_power":   { "value": 180.0, "unit": "W",   "label": "GPU Power" }
  }
}
```

### 6.2 Шаблон отображения

```json
{
  "name": "Gaming",
  "refresh_ms": 1000,
  "pages": [
    {
      "duration_s": 5,
      "rows": ["cpu_temp", "gpu_temp", "fps", "fps_1low"]
    },
    {
      "duration_s": 5,
      "rows": ["cpu_load", "gpu_load", "ram_used", "gpu_power"]
    }
  ]
}
```

| Параметр | Хранение |
|----------|----------|
| SSID / пароль WiFi | LittleFS (`config.json`) |
| IP сервера / порт | LittleFS (`config.json`) |
| Привязка к ПК | LittleFS (`binding.json`) |
| Активный шаблон | получается с сервера при подключении |

---

## 8. Безопасность привязки

- Команды `bind`/`rebind`/`unbind` принимаются ESP **только по UDP от того же хоста**, к которому уже установлено WebSocket-соединение, **или** если устройство свободно
- `pc_id` (UUID) используется как минимальная защита от случайной перепривязки чужим сервером в сети
- Опционально: shared-secret в `config.json` для подписи команд (HMAC)

---

## 9. Нефункциональные требования

| Параметр | Требование |
|----------|------------|
| Задержка обновления данных | ≤ 1с (end-to-end) |
| Время старта ESP в режиме клиента | ≤ 5с |
| Время поднятия AP при потере WiFi | ≤ 30с |
| Интервал обновления данных сервером | 100мс — 5000мс (настраивается) |
| Поддержка одновременных клиентов | ≥ 4 ESP-устройства |
| Интервал UDP beacon | 5с |
| Таймаут активного скана | 3с |
